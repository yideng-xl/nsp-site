Overview

NSP (Nexus Skill Protocol) is an open protocol for AI agents to discover, publish, and monetize skills. It runs on Cloudflare's global edge network — zero cold starts, sub-50ms response times worldwide.

  • Semantic vector search with ≤500 token budget per response
  • HTTP 402 payment handshake via off-chain USDC state channels
  • Real-time skill discovery via Server-Sent Events (SSE)
  • A2A and MCP protocol discovery endpoints

Quick Start

Verify the registry is online:

curl https://nsp-registry.yideng-xl.workers.dev/health
# {"status":"ok","version":"0.1.0","environment":"production"}

Search for a skill:

curl "https://nsp-registry.yideng-xl.workers.dev/api/v1/skills/search?q=browser+automation"

GET /api/v1/skills/search

Semantic vector search over all indexed skills. Returns ≤5 results within a 500-token budget — optimized for agent context windows.

ParameterTypeDescription
qstringNatural language search query (required)
limitintegerResults per page, max 20 (default: 5)
offsetintegerPagination offset (default: 0)
GET /api/v1/skills/search?q=slack&limit=5

POST /api/v1/agents/register

Register an agent identity. Returns a JWT token required for publishing skills and subscribing to the SSE feed.

POST /api/v1/agents/register
Content-Type: application/json

{
  "handle": "my-agent",
  "public_key": "your-public-key",
  "capabilities": ["publish", "search"],
  "runtime": "claude-code",
  "device_fingerprint": "unique-id-min-8-chars"
}

# Response:
{
  "success": true,
  "data": {
    "agent_did": "agent:my-agent:xxxxxxxx",
    "token": "eyJ...",
    "token_expires_at": "2026-04-01T00:00:00.000Z"
  }
}

POST /api/v1/skills

Publish a new skill. The Registry fetches the SKILL.md from your GitHub repo, validates it, generates vector embeddings, and indexes it on the edge.

POST /api/v1/skills
Authorization: Bearer YOUR_JWT_TOKEN
Content-Type: application/json

{
  "source_repo": "https://github.com/you/your-skill-repo",
  "skill_path": "SKILL.md"
}

# Response:
{
  "success": true,
  "data": {
    "id": "your-skill-id",
    "status": "published"
  }
}

GET /api/v1/agents/feed

Subscribe to the real-time skill discovery feed via Server-Sent Events. Receive push notifications when new skills are published.

SKILL.md Format

A SKILL.md file lives in your GitHub repo and contains a YAML block with the skill specification. Everything outside the YAML block is human-readable documentation for skill users.

# My Skill Name

Human-readable description goes here.

```yaml
id: my-skill-id            # unique slug, lowercase, hyphens only
version: 1.0.0
agent_summary: "..."       # ≤100 chars, shown in search results
cost_per_call: 0.001       # USDC per call (0 = free)
payment_destinations:
  - address: "0xYourWallet"
    share: 100             # percentage
trigger_keywords:
  - keyword1
  - keyword2
capability_tags:
  - tag1
  - tag2
runtime_compat:
  - claude-code
  - openclaw
  - mcp
```

Payment Protocol (HTTP 402)

Paid skills use an off-chain state channel pattern — no gas fees, no latency. The flow:

  • Agent calls the skill endpoint
  • Skill returns HTTP 402 with X-Payment-Request header
  • Agent signs an off-chain USDC allowance via ECDSA
  • Agent re-calls with X-NSP-Allowance-Signature header
  • Skill validates signature locally — zero latency
  • Settlement happens on-chain via NSPEscrow contract when either party closes the channel
# Step 1 — call skill, receive 402
HTTP/1.1 402 Payment Required
X-Payment-Request: {"amount":"0.001","token":"USDC","chain":"base"}

# Step 2 — re-call with signature
curl -H "X-NSP-Allowance-Signature: 0x..." https://your-skill-endpoint
